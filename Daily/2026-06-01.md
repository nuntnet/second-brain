# Daily Note — 2026-06-01 (จันทร์)

---

## Key Decisions

- **Brand Web Architecture** — ใช้ Next.js route groups `(brands)/[brand]/subpage` เพื่อให้ URL สะอาด (`/gwm/service` ไม่มีคำว่า "brands" ซ้ำ)
- **BrandSubNav Pattern** — sticky tabs ใต้ main navbar ทุก brand sub-page; brand logo fade-in ใน main navbar เมื่อ scroll ผ่าน hero (IntersectionObserver)
- **Scroll Behavior** — sub-pages โหลดมาแล้ว scroll ข้าม hero ไปเลย — ผู้ใช้ไม่ต้อง scroll หาเนื้อหา
- **Notion DB Schema** — ใช้ Notion REST API ตรงๆ (`PATCH /v1/databases/{id}`) แทน MCP tool เพราะ MCP write ติด permission บน Private DB บ่อย
- **Cloudinary Upload Optimization** — ฝัง transform params (`f_auto,q_auto:good,w_1920`) ใน `cloudinary-upload.ts` ตอน upload ทันที แทนที่จะแก้ทุก component

## Things Learned

- Next.js route group `(brands)` — วงเล็บ = ไม่นับเป็น URL segment (เป็นแค่ folder จัดระเบียบ code)
- Notion MCP `update-data-source` ติด 404 บน Private DB — direct Notion API ผ่าน `NOTION_API_KEY` ทำงานได้ทันที
- `bun run dev` ของ ch-erawan-next ใช้ `rm -rf .next` ทุกครั้ง — ถ้ารัน concurrent process จะ webpack corrupt ต้อง restart สะอาด
- Lighthouse CLI ติดตั้งอยู่แล้ว (`/Users/nunt/.bun/bin/lighthouse`) — ใช้วัด performance ได้ทันทีโดยไม่ต้อง API key
- HTML ของ ch-erawan homepage จริงๆ 36KB หลัง brotli (ไม่ใช่ 283KB raw)
